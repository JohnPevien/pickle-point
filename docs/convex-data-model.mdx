## Decision

Use Convex for persistence, backend functions, and realtime sync.

Remove Turso and Drizzle from the target architecture.

## Core Tables

- Auth identity mapping.
- Game Master workspaces.
- Venues or clubs.
- Players.
- Open play sessions.
- Session players.
- Session queue counters.
- Session matches.
- Match history.
- Tournaments.
- Tournament entrants.
- Tournament matches.
- Stats snapshots.

## Realtime Use

Use Convex subscriptions for live player views and Game Master dashboards.

The most important realtime surfaces are:

- Open play Game Master control.
- Open play public live view.
- Tournament admin control.
- Tournament public bracket view.

## Operational State

High-churn session state is stored in session-specific tables instead of embedding unbounded arrays in parent records.

Examples:

- `sessionPlayers` stores queue status, playing status, paused status, sit-out counters, and rotation metadata.
- `sessionMatches` stores active and completed open play court assignments.
- `tournamentMatches` stores bracket match slots, source match references, scores, and correction state.
