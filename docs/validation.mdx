## Documentation Changes

Run:

```bash
pnpm lint
pnpm build
```

Acceptance:

- `/docs` builds successfully.
- `docs/meta.json` includes any new MDX pages.
- `docs/task-log/` includes task status changes when work starts, completes, or creates follow-up tasks.
- No duplicate docs source exists outside root `docs/`.
- `docs/README.md` stays useful for future AI agents.

## Feature Changes

Run:

```bash
pnpm test
pnpm lint
pnpm build
```

Acceptance:

- New or changed backend behavior has Convex tests.
- UI business logic has focused tests.
- Existing route and helper tests still pass.
- User-facing docs are updated.
- The relevant task log page is updated with validation evidence.

## Local Git Hooks

Husky installs local Git hooks through the `prepare` script.

Pre-commit runs:

```bash
pnpm check:commit
```

This currently runs `pnpm lint` so commits get fast feedback before they are created.

Pre-push runs:

```bash
pnpm check:push
```

This currently runs `pnpm test`, `pnpm lint`, and `pnpm build` so pushed branches match the feature-change validation contract.

These hooks are local developer guardrails. They do not replace manually running the relevant validation commands before marking a task done, and they do not replace CI when CI is added.

## Convex Changes

Before editing Convex code:

- Read `convex/_generated/ai/guidelines.md`.
- Keep functions validated with `v`.
- Use indexes instead of query filters where possible.
- Keep auth decisions based on `ctx.auth.getUserIdentity()` and `identity.tokenIdentifier`.
- Avoid unbounded arrays on parent documents for high-churn data.

## Task Sync

ClickUp is retired as the active task system. Use `docs/task-log/` for task status, follow-ups, acceptance criteria, and validation evidence.
