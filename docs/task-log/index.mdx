## Purpose

This task log is the canonical execution queue for Pickle Point. It replaces ClickUp for active planning, status tracking, completion evidence, and future agent handoff.

Use this log before starting work, during handoff, and when reconciling stale planning sources. ClickUp links are preserved only as migration provenance.

## Source Of Truth

Use this order when task status sources disagree:

1. Current repo implementation and tests.
2. This task log.
3. `PRD-NEW.md`.
4. Imported ClickUp task status.

## Status Rules

- `Pending`: accepted follow-up that is not started.
- `In Progress`: current or partially complete work that still needs implementation, tests, docs, or validation.
- `Done`: implemented, decided, documented, or intentionally retired.

The task log tracks planning state, scope, acceptance criteria, validation, and completion evidence. `docs/changelog.mdx` tracks shipped or meaningful project changes. Link a completed task to the changelog only when user-facing behavior, architecture, data model, dependency policy, validation policy, or accepted risk changed.
