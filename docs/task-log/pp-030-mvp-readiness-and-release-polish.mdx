## Known Gaps / Limitations

- Player login account claim/link flows are deferred; MVP participation remains based on local player records managed by the Game Master.
- Duplicate player records across Game Master workspaces are not auto-merged in MVP.
- Singles tournaments, random pairing/spin-the-wheel social play, and AI-assisted matching are post-MVP features.
- Custom OTP, resend cooldowns, and app-owned email verification are deferred to WorkOS AuthKit's hosted authentication flow for MVP.
- Additional code splitting beyond the QR renderer is deferred until measured bundle or performance issues justify it; route-local tables and panels are kept inline for MVP simplicity.
- The current MVP readiness record is documentation and validation state only; it does not replace hands-on QA with real Game Master workflows.
