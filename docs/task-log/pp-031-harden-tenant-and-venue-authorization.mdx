## Status

Done

## Goal

Apply explicit public, owner, and Game Master boundaries to tenant and venue Convex functions without trusting browser-supplied tenant authority.

## Acceptance Criteria

- Public tenant lookups return active, contact-free branding projections.
- Workspace settings and private tenant data require an active owner identity.
- Venue listing and CRUD require an active owner or Game Master identity.
- Venue update and delete authorization derive the tenant from the loaded venue.
- Unauthenticated, player, suspended, and cross-tenant access fails closed.
- The Convex access matrix records the hardened behavior.
