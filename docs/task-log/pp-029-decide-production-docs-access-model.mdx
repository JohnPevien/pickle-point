## Goal

Decide whether production docs should be public, admin-only, or deployed separately.

## Acceptance Criteria

- Review the intended audience for production docs.
- Choose public, admin-only, or separate deployment.
- Document the chosen access model in `docs/decisions-log.mdx` and `docs/architecture.mdx`.
- Add implementation work only if the decision requires route protection or deployment changes.

## Decision

Production docs are authenticated when WorkOS is configured and unavailable in production when WorkOS is incomplete. Local development may keep docs public when WorkOS is not configured.
