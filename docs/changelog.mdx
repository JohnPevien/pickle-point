## 2026-06-15

- Added `docs/task-log/` as the versioned task source of truth, replacing ClickUp for active Pickle Point task tracking.
- Imported Pickle Point ClickUp task provenance into per-task MDX pages and kept only the remaining test coverage work in progress.
- Documented that the task log tracks execution state while the changelog tracks shipped or meaningful project changes.
- Moved living documentation source to root `docs/`.
- Flattened docs navigation into root-level MDX files.
- Added architecture, roadmap, task contract, validation, changelog, and acceptance criteria docs.
- Added `docs/README.md` for future AI-agent context.

## Recent Merged Work

- Phase 3.1 hardened tenant and venue authorization: public tenant queries now return active, contact-free projections; workspace settings require an owner; and venue management requires an owner or Game Master with resource-derived tenant checks.
- README, PRD, and roadmap state were reconciled against the current MVP implementation and task-log source of truth.
- Production docs access was defined and enforced: local docs remain available for development, while production docs and docs search require WorkOS auth when configured.
- Admin navigation integration added a shared admin shell and links for dashboard, open play, tournaments, players, venues, and workspace.
- Auth and Phase 3 test coverage added WorkOS route tests, auth helper tests, open play helper tests, and tournament helper tests.
- Product decision docs expanded player model, tournament, matching, and auth decisions.
- Tournament seeding and corrections added seed editing, seed validation by skill tier, downstream correction behavior, and double-elimination reset final behavior.
- Open play fair rotation added sit-out metadata, rotation priority, visible sit-out state, and backend tests.
- Manual match adjustment added court rename, player swap, substitution, cancellation, and guardrail coverage.
- Tournament UI added list/detail management, admin score entry, public bracket views, and QR/live-link support.

## Sync Notes

`docs/task-log/` is the active status source. `PRD-NEW.md` is retained as historical product context and now includes a current implementation status note for stale-checklist interpretation.
